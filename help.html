Introduction:

      Metabolic Interactive Nodular Network for Omics (MINNO) is a D3 javascript library based web application to visualize biological data onto metabolic networks. Besides visualization, this tools offer various ways to share data and results to facilitate collaboration across multiple research groups globally. Being a web-application, it does not require any installation and it can be run on both Chrome and Mozilla browsers. MINNO comes with 66 base metabolic pathways from the KEGG database, which cover all primary metabolic pathways and can be combined together to build any possible large-scale metabolic network as needed. Beside KEGG, the tool also has access to the metabolic network models from BiGG Models database. MINNO enables users to built organism specific metabolic map by superimposing large-scale metabolic network formed from KEGG database with organism’s known metabolic pathways data without need to rebuild a network from scratch for each organism. The BiGG models already comes with organism specific models, so need to superimpose any metabolic maps. However, user need to organize the network as BiGG models do not come with prelayout network maps.

The tool accepts any kind of omics data, including metabolomics, proteomics, and fluxomics data as long as they can be visualized by changing sizes of nodes and edges of the metabolic network. The modular design promotes accelerated investigation of omics data on genome-scale metabolic networks for multiple species.

Databases:

MINNO comes with two biological databases: KEGG (Kyoto Encyclopedia of Genes and Genomes) and BiGG (Biochemical, Genetic and Genomics) databases.

  1. KEGG-Database: KEGG (Kyoto Encyclopedia of Genes and Genomes) is a knowledge base for systematic analysis of gene functions, linking genomic information with higher order functional information stored in the PATHWAY database, which contains graphical representations of cellular processes, such as metabolism, membrane transport, signal transduction and cell cycle. The KEGG database harbour metabolic pathways of 860 eukaryotes and 7,934 prokaryotes, of which 410 are archaea making it an excellent database to explore cross-species interaction in diverse environmental conditions and discover unknown links and interactions between biochemicals or species. MINNO contains 66 pre-layout metabolic pathways for quick use and data visualization and about 150 in total metabolic pathways for metabolism. Remaining non-metabolic pathways cannot be currently visualized on the tool. The menu contains two categories: 1) selected Ortholog/base maps which contains 66 pre-layout metabolic pathways 2) list of pathways which contains all the metabolic pathways. User can also access all the pathways documented for specific user selected organism and superimpose on base maps to obtain organism’s specific metabolic network.

  2. BiGG-Models: BiGG Models database is a knowledgebase of genome-scale metabolic network reconstructions. BiGG Models integrates more than 70 published genome-scale metabolic networks into a single database with a set of stardized identifiers called BiGG IDs. Genes in the BiGG models are mapped to NCBI genome annotations, and metabolites are linked to many external databases (KEGG, PubChem, and many more).

Tutorial:

This tutorial showcases a sample dataset containing intracellular, extracellular, and flux metabolomics data from two organisms: Staphylococcus aureus (SA) and Pseudomonas aeruginosa (PA). Throughout this tutorial, you will gain knowledge on downloading a basic network pathway from the KEGG database, incorporating supplementary reactions into the network, and uploading a dataset onto your customized network. Download sample dataset

  • Part 1: Downloading the ortholog network:

    1. Download the sample dataset and save in a new folder.

    2. Click the "Simulation ON" button, which will update to display "Simulation OFF" that will retain the (X,Y) coordinates of the network layout. To access network files, click on the drop-down menu labeled 'KEGG Database'.



    3. Enter 'Arginine biosynthesis' in the search bar and choose 'Arginine Biosynthesis' from the Selected Ortholog Maps category.



    4. Click on the Select Rxns drop-down menu and click 'Select All'.



    5. Now, the Arginine Biosynthesis network should be visible. You can adjust the position of the network on the screen by dragging it with your cursor. Additionally, you can zoom in or out using the scroll function on your mouse.



    6. To export the network in JSON format, right-click on the screen. From the options that appear, select "Export," then choose "Network," and finally click on "JSON." Save the exported file in JSON format in the same folder as the sample dataset. This is an ortholog or base network file. It is important to save the file with the prefix "Ortholog_ko_". MINNO tool will automatically suggest a suitable name for the file, and it is recommended to use the suggested name. If the file name already exists, you can add a convenient suffix to differentiate it.



  • Part 2: Downloading the organism network:

    1. Now, click on the "KEGG Organisms" list drop-down menu. Search for "Pseudomonas aeruginosa" and select PA01 strain.



    2. After selecting the specific species strain (PA01), we need to specify the metabolic pathway. Click on the "KEGG Database" drop-down menu and search for "Arginine Biosynthesis". Under the "List of Pathways" category, select "Arginine biosynthesis".



    3. A new window will pop up with an XML file. This is the KGML file from the KEGG database titled "Arginine biosynthesis" of organism "pae", which is the KEGG ID for Pseudomonas aeruginosa. Right-click on the file and select the "Save as" option to save the file in the same folder as the other files.

    4. Now, upload the XML file that you just saved. If the "Simulation ON" option is active, the nodes of the network will self-organize based on the D3.js library. If not, the network will remain static. Right-click on the screen and select "Export > Network > JSON". This is an organism-specific KEGG file. The file name should have the prefix "KEGG_org_" where "org" represents the organism ID. MINNO tool will automatically suggest a suitable name for the file, and it is recommended to use the suggested name. If the file name already exists, you can add a convenient suffix to differentiate it. Save this file in your tutorial folder.



  • Part 3: Combining the ortholog and organism specific network/s:

    1. Refresh MINNO, ensuring that the simulation is off to prevent any alterations to the network layout due to node movements. Right-click and select Import > Network. Choose both the Ortholog and KEGG networks you created in the folder (in .json format only). Wait for the reaction menu to update with the newly uploaded files. Click on Select Rxns > Select all, and the network will appear on the screen. All the annotated reactions for the PA01 metabolic network in the KEGG database will be displayed as nodes connected by black lines/edges, while unannotated reactions will be represented by light grey lines/edges between nodes. Right-click to export this file in JSON format as before, and save it as "KEGG_pae_Arginine biosynthesis_combined.json" in your tutorial folder. You can choose any arbitrary name as long as it allows you to identify it as the Arginine biosynthesis network of Pseudomonas aeruginosa.



  • Part 4: Adding reactions to the network:

    1. In case you need to add a reaction for the transport of extracellular metabolites such as L-Arginine, right-click on the screen and select Insert > Reaction.



    2. To view the reaction node name, click on the Grid, Tooltip drop-down menu and unselect "Reaction node name." This will display the reaction names.



    3. Double-click on the text beside the new reaction nodes to change their names. For the grey reaction node, change the name to "L-Arginine outflux," and for the white metabolite node, change the name to "L-Arginine." You do not need to change the Subsystem or gene names, but you can add them to your future visualizations if desired.



    4. Click and drag to re-arrange the reaction node so that the white metabolite node is on top of the reaction node. For alignment, press the SHIFT key and drag your cursor over the nodes from L-Arginine down to N-acetyl-L-citrulline in the original network. Once those nodes are all highlighted in green, release the SHIFT key and right-click. From the menu, select Geometric operations > Alignment > Vertical. This will align the selected nodes vertically. If you make a mistake with the alignment, you can use the right-click menu to select the Undo option.



    5. To connect your new outflux reaction to the rest of the network, hold down the CTRL key and left-click on the L-Arginine node of the original network. Then, while still holding CTRL, drag your cursor to the L-Arginine outflux grey reaction node and release the mouse button. A black line connecting the two nodes will appear. If you accidentally drag over some text while performing this step, you may need to repeat the process to ensure the connection is made correctly.



    6. Export this new network by right-clicking on the screen and selecting Export > Network > JSON. Save the file in your tutorial folder with the name "KEGG_pae_Arginine biosynthesis_combined_rxn.json". After saving, refresh your browser and click on the Simulation ON button to preserve the network layout. Next, re-upload your new network file by selecting Import > Network from the right-click menu. Choose the JSON file you just saved and then select all the reactions under the Select Rxns menu. The network with your outflux reaction should now appear. Note that by default, only the metabolite node names are displayed, but you can un-select "Reaction node names" on the Grid, Tooltip drop-down menu to view or edit the reaction names.



  • Part 5: Uploading data to the network:

    1. Right-click on the screen and select Import > Datafile. Choose the tutorial datafile that you saved at the beginning. To classify different metabolite nodes based on their groups, press and hold the SHIFT key, and then drag your cursor over the intracellular nodes, excluding the recently added L-Arginine node. Once the desired nodes are highlighted in green, right-click on the screen and select Select > Node from the menu.



    2. Click on the Group drop-down menu, which is currently displaying "extracellular". Select "intracellular" from the menu. Then, left-click on the network screen to deselect those nodes. Repeat the same process for the extracellular node, which is the L-Arginine node that you added manually. Make sure to only highlight the white metabolite node, not the grey reaction node. From the Group drop-down menu, select "extracellular".



    3. To normalize the data for each group, click on the Methods drop-down menu. It should currently be displaying "None". Then, click on "Control Normalization" under the extracellular subheading.



    4. Next, click on the Parameters drop-down menu and select "scaling". Drag the larger slider bar to your desired scaling value, in this case, 2. Repeat this process for the intracellular subheading by selecting "Control Normalization". To register the scaling factor, slide the bar away from 2, release it, and then slide it back to 2 again. Remember to adjust the parameters for each group separately. Under the Link Group section, select "None". For the scaling value, adjust it to 1 as these data are already normalized and do not require additional normalization steps.

    5. Now, click on "Invert" on the color bar at the bottom of the menu. This will make blue color represent low values while red color represent high values. Please note that for your own visualizations, you can choose any color scheme from the color-scheme drop-down menu that suits your preferences.



    6. To visualize the Staphylococcus aureus (SA) data, click on the menu at the top of the screen that currently displays "PA" and select "SA" instead. The nodes on the screen that have SA data should now be colored accordingly, and the outflux reaction should have a bolded arrow indicating the flux. You can follow the same instructions to access the S. aureus data on a different tab in your browser and display the data there. This way, you can compare the data from both species at each time point.



    7. To change the timepoint to 6hr, select the menu that currently displays "12hr" and choose "6hr" instead. This will update the visualization to show the data for the selected timepoint. For cleaner final visualizations, you can select "Link node" from the Grid, Tooltip drop-down menu. This will remove the grey link nodes from your network, while still keeping the white metabolite nodes visible. This can enhance the clarity of your network visualization.



    8. To practice some of these features, you can try adding another reaction to the network. First, refresh your browser to remove any existing data from the network. Then, follow the same steps you used to add the L-Arginine outflux reaction to add an L-Glutamine outflux reaction. This will allow you to explore the process and familiarize yourself with adding reactions to the network. It's worth noting that this program contains many more features beyond the scope of this basic tutorial. If you're interested, I recommend reading through the instruction manual to discover additional features that can further customize your networks and enhance your experience with the tool.



Trouble-shooting Refresh your browser We recommended to save many versions of your network as you make changes and then refresh/upload the new version to avoid losing network data and to decrease the lag time when adding new features to the network.

Datafile Schema:

The datafile can be either .csv or .json format to be loaded into the visualization code. Each datafile format have information of variables/conditions which are varied and quantitative measurements of various biomolecules which are represented as nodes or edges in the network. Below are the steps to make datafile:

  1. Assuming three variables which are varied to be ‘X’, ‘Y’, ‘Z’ where 'X', 'Y' and 'Z' can be any arbitrary variables such as organism name, chemical treatment etc. If there are no three parameters for any experiment data, then just define arbitrary variable and give a constant value to it. Here is what schema the datafile must follow.
  2. JSON: Assuming same three variables which we have used above, the data should be organized as shown below:
    {‘X1’:
    {‘Y1’:[‘name’: ___, ‘gp’:____, ‘Z1’:___, ‘Z2’:____, ‘Z3’:____ .....],
    ‘Y2’:[‘name’: ___, ‘gp’:____, ‘Z1’:___, ‘Z2’:____, ‘Z3’:____ .....],
    .
    .
    .
    }
    ‘X2’:
    {‘Y1’:[‘name’: ___, ‘gp’:____, ‘Z1’:___, ‘Z2’:____, ‘Z3’:____ .....],
    ‘Y2’:[‘name’: ___, ‘gp’:____, ‘Z1’:___, ‘Z2’:____, ‘Z3’:____ .....],
    ..
    .
    },
    .
    .
    .
    }


    Above X1, X2, Y1, Y2, Z1,Z2... are different X, Y, Z variables respectively as there can be arbitrary number of these variables in a given dataset. In addition to these three variables, we also have ‘name’ to specifically locate node in the network and ‘gp’ for group of dataset that one wants to be loaded onto the node. This allow same node name to represent multiple dataset depending on what it represents in a given network. For instance, if one is trying to visualize glucose uptake by E.coli, then one need to specify extracellular and intracellular signal intensity for glucose and how fast signal intensity is changing provide flux value. This way, same glucose node in the network can show different dataset. Generally biological experiments have control experiment which provides a way to compare and relate how certain perturbation has impacted the experimental results. This control dataset is also need to be included in the above both datafile formats and must be represented as ‘C’ in the datafile as a variable that is varied during the experiment.
  3. CSV: CSV format is easier to create using Microsoft Excel or Libre Office. The csv file must have following header

    Compound, Group, X1_Y1_Z1, X1_Y1_Z2,....
    A , b, 1.0, 2.0 ....
    .
    .


    Here, X1_Y1_Z1 represents three variables as a single string which are tied together using underscore character. Each line in the csv file has a compound name, its group id for example whether the compound represents "I" (intracellular) or "E" (extracellular) molecule etc, and quantitative measurement for X, Y and Z variables. Except for first two column, the rest of the header does not need to be in any particular order as long as all of them are included. Generally biological experiments have control experiment which provides a way to compare and relate how certain perturbation has impacted the experimental results. This control dataset is also need to be included in the above both datafile formats and must be represented as "C" in the datafile as a variable that is varied during the experiment.


Visualization

Load datafile and its visualization onto the network: To load datafile and visualize onto the network, follow following steps:


  1. After loading the network file load datafile and then user must assign group to nodes before normalization. Select Assign group menu and select specific type of nodes and assign all the node to desired group from the menu.

  2. Then comes applying methods. Each group of dataset needs to be uploaded onto the network can have different normalization method. Often raw data contains big number which cannot be loaded onto the network. Thus, normalization and scaling is required. Select suitable method for specific group and adjust scaling parameter value using the big slider in the middle. Do this for all groups of data whether it is intracellular, extracellular or edges of the network.

  3. It is done. Now change the datagroups from the top 3 menus which update the network as per values for 3 variables you provided in the datafile.

Data Normalization: The data normalization methods appears in the "Method" drop down menu after datafile is loaded. MINNO allow user to define groups to different nodes of the network. Subsequently, data from the datafile will be expressed onto nodes as per their group defined by the user. Each group from the datafile can be treated individually depending on which method is suitable for them. Four normalization methods are provided by the tool:

  1. None: This method allow data values to be loaded and used as they are in the datafile onto the network nodes. No normalization method is implemented in this option.

  2. Control normalization: This method computes the standard deviation of the control dataset for each compound. Then it subtract the value measured in certain condition with the value in control case for each metabolite and specific time point and divide the difference with the standard deviation to get normalized value.

    Let std_dev be the standard deviation of the control dataset. Then the normalized data can be obtained using below eq.

          Ny(t) = (y(t) – c(t))/std_dev

    here Ny(t) is the normalized value of y(t) from Y dataset at time t when it is compared with the control value via c(t) at the same time.

  3. Control Mean normalization: This method computes the mean and standard deviation of the control dataset for each compound. Then it subtract the value measured in certain condition at specific time point and the mean value of the control dataset and divide the difference with the standard deviation to get normalized value.

    Let c_mean and std_dev be the mean and standard deviation of the control dataset. Then the normalized data can be obtained using below eq.

          Ny(t) = (y(t) – c_mean)/std_dev

    here Ny(t) is the normalized value of y(t) from Y dataset at time t when it is compared with the mean value of the control dataset C for all time point.

  4. Log2 fold change: This method computes log(y(t)/c(t)) value for Y dataset at time t with respect to control dataset c(t) at time point 't' for each compound.

Parameter: By default the tool use constant base value and allow user to decide the scaling value to be used for any given group within the datafile. The scaling value essentially allow how much one unit difference in the normalization data would translate into node sizes. Scaling value >1 means that the 1 unit difference in the normalized data would translate into >1 unit difference in the node size. Such value of scaling to be used when the variation in the normalized data values is small and hard to distinguish visually. On the other hand it is possible that the datafile contain values which have large variation such that everything takes extreme values i.e., nodes appears either giant or point size. In such situation, we need to use scaling <1 to diminish the variation is certain group/s and focus on the relevant ones which do have extremely large or small values. The value of the scaling parameter can be adjusted by using the bigger slider above PNG download button. Each time slider value is adjusted, normalization of only selected group changes. Keep doing for all the different groups you have in the datafile. This way user can normalized wide variety of data specifically.

Assign Group: This menu contains options for different group that can be assigned to network nodes. This menu is updated when datafile is uploaded in the web-application. To assign specific group to any node, one first need to select all nodes which belongs to the specific group such as intracellular or extracellular and then select the group from the Assign group drop-down menu. After assigning different groups to nodes, user must select the method to each group and scale the values using scaling option. At first user must select scaling to be 1. Later user can change its value depending on need.

Additional features

It offers multitudes of features which are listed below:

Simulation: This key will allow self-organization of nodes in the network based on force simulation where nodes in the network repel each other while edges attract them together. This is useful when network does not come with initial coordinates such as metabolic networks from BiGG-models database However, if the network is already pre-layout such as network from KEGG database, then keeping simulation switched off will make network nodes static.

Freeze: This feature is added to improve user experience while using the tool in case when the network are significant big. However, having too big files are not recommended as it can crash the browsers. This web-application uses D3 JavaScript library as underlying program to layout the network visualization. In case of big files, this can take long time to execute and user actions will changes/updates the visualization slowly. By clicking it on, user can stop the underlying computation early hence improving the user experience. But this also comes with a cost as it prevent dragging of network elements jerky or even make them hard to move and some part of network may not even update completely.

Three data groups: These three drop-down menu will be containing 3 variables from the datafile. These 3 variables are variables which can be varied within an experiment or processed data. For example, time is one commonly used variable in any dataset. The other two variables could be two kinds of treatments or different Omic data and one is interested in combination of these two variables together. Thus, having 3 variables for this tool expand its application to larger variety of experiments. The datafile can be either csv file or json file. Click here for more details.

KEGG organisms: This drop-down menu offers list of all organisms listed in KEGG database. To obtain species specific metabolic pathway, select all the organisms for which you need metabolic pathways. Then select the metabolic pathway from the "KEGG database" menu under "list of pathway" category.Click here for list of all organisms.

Select Rxns: Once a network file is loaded, the list of reactions will appears here. User can select all or specific ones by scrolling over the list.

Remove element:This menu contains list of nodes sorted based on their degree in the network. By selecting elements from the menu, these elements will be removed globally from the metabolic network.

Repeat element: This menu contains list of nodes sorted based on their degree in the network. By selecting elements from the menu, these elements will be duplicates globally from the metabolic network.

Find element: This menu contains list of nodes in the network sorted alphabatically. User can use this list to find specific elements in the network.

In the menu right to the Assign-group menu, there are few more features as discussed below:


  1. Grid: This feature shows/hides the grid in the visualization. The purpose of grid is to offer way to spatially organize network via combing different network files.

  2. Tooltip: This feature shows details associated with individual elements of the network. For KEGG metabolic networks, this also offers more options to explore metabolite, rxn and gene details from the KEGG database.

  3. Reaction node name: This feature show and hide the name of reaction nodes of the metabolic network. These reactions nodes also called as link nodes connecting various metabolite nodes together to represent a chemical reaction.

  4. Link node: Show or hide the link node.
Attribute Menu: This provide all the options that user can vary as desired. They are grouped into text, rectangle, spacing, node,link, image resolution and freeze degree. User can vary various attributes by adjusting smaller slider below the menu. On either side of slider, there are inputs where user can provide values to adjust the range of the slider for different attributes as required. There is a limit of +-1000 units value. Anything above or below absolute 1000 units will set default to 1000 value. User can select network elements on which such modifications can be implemented by selecting them by either clicking on them individually or using shift key with mouse drag to scan the area interested.

  1. Text: Provides options for changing font-size, x and y position of text relative to node position.

  2. Rectangle: Provides options for changing width, height, border thickness, corner roundness of the selected rectangles.

  3. Spacing: Provides options for changing linear spacing, circular spacing between elements of network.

  4. Node: Changes the size of the node.

  5. Link: Changes the thickness of the link/edges.

  6. Image resolution: Allow user to vary image resolution for the png image download as desired. While if there is a image imported into the visualization then option to change the dimension of image file while maintaining the aspect ratio of the selected image the same.

  7. Freeze degree: This option control how much internal computation for simulation or user interaction such as dragging user want to compromise. Magnitude closer to 1 prevents all the dragging of network elements. More precisely, element can still move but their new position will not update immediately. However, when user want to move some other element, previously moved elements will have new position. If magnitude value is reduced to 0.5-0.3, user can get dragging behaviour but may not be as smooth as you will have if you choose magnitude value to 0.01. As you will notice with big files, having very small value for freeze make everything very slow as internal computation takes time to update changes user made. Thus this features improve user experience by avoiding unneccesary computation.

Color Scale Menu: D3 library provides several sequential, diverging color scales with single or multiple hues. Click here for more detail.

Invert: Invert the color scale.

Color scale: This shows the color scale chosen from the Color Scale Menu. It also offers a thin black strip which can be dragged using mouse. User can select network element and set the color by dragging the black strip.



Edit tools

In addition to above features, the web-application has another menu which pop-up on mouse right click on the visualization screen. It offers options like file upload to download into various formats and various geometric operation to make organization of network elements easier and faster.

Below we provide the list of options provided in the pop-up menu:

  1. Insert:

    • Text: Add arbitrary text to the visualization to add necessary information. The font size and color can be adjusted using attribute Menu discussed above.

    • Reaction: Add two nodes connected by a directed edges.

    • Node: Add a node element of the network. Note: User cannot connect nodes together and build the network as the tool will exclude such connections while uploading network file. Node must connect to link node and vice versa.

    • Link node: Add a link node of the network. This allow connecting two network elements. Note: User cannot connect nodes together and build the network as the tool will exclude such connections while uploading network file. Node must connect to link node and vice versa.

    • Rectangle: Add a rectangle to the visualization. Its attributes such as border thickness, roundness and color can be adjusted using attribute menu discussed above.

    • Color scale: Add color scale which appears on the bottom of the taskbar. This scale allow interpretation of data when loaded onto the network.

  2. Select:

    • Node: Select only nodes which were scanned using shift key.

    • Link node: Select only link nodes which were scanned using shift key.

    • Link: Select links connected to selected nodes which were scanned using shift key.

    • Both nodes and links: Select both nodes and links which were scanned using shift key.

  3. Delete:

    • Only selected: Delete only selected network element/s or other elements created in the visualization.

    • All: Delete everything on the visualization screen.

    • Color scale: Delete color scale which can be added using insert option.

  4. Geometric Operations:

    • Alignment:

      • Horizontal: Horizontal alignment of selected network elements. User can use this option to align selected nodes, rectangles or images used in the visualization.

      • Vertical: Vertical alignment of selected network elements. User can use this option to align selected nodes, rectangles or images used in the visualization.

      • Circular: Circular alignment of selected network elements. This form a equi-spaced ring. It is important to select network elements that user wants to appears as a ring.


    • Layers: This web-application creates various svg elements and group them in classes. For examples, the nodes are grouped under nodes group, links are grouped under links group, rectangles are group under rectangles group etc. How and when certain group is created decides which elements will come on the top of others. However, user may prefer to reorder these groups to built more appealing visualization. To do that, the application offers way to push different group at the top or bottom.

      • Bring to front: Bring entire group of elements at the top.

      • Push it back: Pushes all the elements of the group at the bottom.

      These options changes the order of groups but not the order of elements within individual group. This way user can push rectangle svg group at the bottom so that the network can be visualized on the top of it. However, when multiple rectangles are created, user need to manage how different rectangles will order themselves based on order of their creations.

    • Duplicate: This option allow user to make duplicates of selected network elements. Using this, certain part of the network can be duplicated and its nodes and links can be colored or change their size to highlight different group of data. This feature is basically to create a simplified network representation to compare different datasets.

    • Rotation: Rotate selected network elements about their centeriod.

    • Make arcs: Make curved edges. For example, it makes nice curved edges for circular loops in the network.

    • Remove arcs: Revert back curved edges to straight.

    • Highlight: Change opacity to 1.0 to selected network elements. This feature also allow user to select which reactions will be selected to be included for the KGML file of the network. Only nodes and edges whose opacity is 1.0 will be included in the KGML file.

    • Un-Highlight: Change opacity to 0.5 to selected network elements. This feature also allow user to select which reactions will be selected to be included for the KGML file of the network. Nodes and edges whose opacity is 0.5 will be excluded in the KGML file.

  5. Undo: It only undo nodes movements or dragging event by the user. If you delete or insert any network element, Undo cannot revert to the previous network.

  6. Redo: It only redo nodes movements or dragging event by the user. If you delete or insert any network element, Undo cannot revert to the previous network.

  7. Help: Opens up User Manual.
